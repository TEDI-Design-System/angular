# Printing

## Components have a built-in styles for printing

- Text color and some components are converted into grayscale
- Most background colors are set to white
- Base font size is 14px and all other font sizes and spacing is adjusted accordingly
- Some component are hidden by default (Buttons, Anchors, Header, Footer, Breadcrumbs, Sidenav)
- Expandable components are open by default (Accordion, Collapse, Table subRow/subComponent)
- Prevent page break in some components (Placeholder, Table rows). Rest of the page break locations should be defined in the app using `<Print>` component. <br/>
  <i>
    <b>NB! Page break does not work in Firefox. </b>
    [Bug 1](https://bugzilla.mozilla.org/show_bug.cgi?id=1807406), [Bug 2](https://bugzilla.mozilla.org/show_bug.cgi?id=1695475),
    [Bug 3](https://bugzilla.mozilla.org/show_bug.cgi?id=939897)
  </i>
- Tables are printed by default. Every table should be reviewed case by case. When table does not fit horizontally we have a scrollbar. This means that tables with many columns might not fit in the print view.

<p>
  <b>UI can be adjusted using the following methods:</b>
</p>

### `<Print>` component

Preferred method. [More info here](/docs/components-print--docs).

### Hooks

- `const isPrinting = usePrint()` - Checks if `window.matchMedia(print)` matches. Useful when you want to conditionally re-render UI during printing.

### Utility classes

<p>
  <b>NB! Only use utility classes when usage of `<Print>` component is not possible.</b>
</p>

- `no-print` - Hide component from print view
- `show-print` - Can be used to show components in print view, that are hidden by default
- Determine where page breaks during printing. Value can be `auto`, `avoid`, `avoid-column`, `avoid-page`, `avoid-region`
  - `break-before-{value}` - How page, column, or region breaks should behave <b>before</b> a generated box
  - `break-after-{value}` - How page, column, or region breaks should behave <b>after</b> a generated box
  - `break-inside-{value}` - How page, column, or region breaks should behave <b>inside</b> a generated box
